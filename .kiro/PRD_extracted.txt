7. Sistem Reseller
7.1 Syarat Join
Top up minimum saldo (default: Rp 18.000, dapat dikonfigurasi admin)
Evaluasi bulanan otomatis; gagal → downgrade dengan notifikasi
Admin dapat trigger evaluasi manual via menu Reseller

7.2 Keuntungan Reseller
Akses server reseller (harga lebih murah dari harga user biasa)
Fitur lock/unlock akun VPN milik pembeli
Statistik transaksi bulanan
Markup harga independent: VPN, PPOB, Akrab, SMM
Harga beli = harga API + markup global admin; harga jual = harga beli + markup reseller

7.3 Alur Evaluasi Reseller Bulanan
Sistem cek stats reseller pada awal bulan baru
Jika total akun < min_accounts ATAU total top up < min_topup → status dicabut
Notifikasi dikirim ke reseller H-7 sebelum periode berakhir
Admin dapat restore reseller yang dicabut via menu Restore Reseller

11.3 DDL Tabel Baru (SQL)
-- smm_orders
CREATE TABLE IF NOT EXISTS smm_orders (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  order_id TEXT,
  service_id INTEGER,
  target TEXT,
  quantity INTEGER,
  amount INTEGER,
  status TEXT DEFAULT 'pending',
  created_at INTEGER,
  FOREIGN KEY (user_id) REFERENCES users(user_id)
);

-- akrab_orders
CREATE TABLE IF NOT EXISTS akrab_orders (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  reff_id TEXT UNIQUE,
  produk TEXT,
  tujuan TEXT,
  amount INTEGER,
  status TEXT DEFAULT 'pending',
  created_at INTEGER,
  FOREIGN KEY (user_id) REFERENCES users(user_id)
);

-- ppob_orders
CREATE TABLE IF NOT EXISTS ppob_orders (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  order_id TEXT,
  product_code TEXT,
  target TEXT,
  amount INTEGER,
  status TEXT DEFAULT 'pending',
  created_at INTEGER,
  FOREIGN KEY (user_id) REFERENCES users(user_id)
);

-- markup_config
CREATE TABLE IF NOT EXISTS markup_config (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  scope TEXT NOT NULL,       -- 'global' | 'reseller'
  service TEXT NOT NULL,     -- 'smm' | 'akrab' | 'vpn' | 'ppob'
  user_id INTEGER,           -- NULL = global
  type TEXT NOT NULL,        -- 'pct' | 'flat'
  value REAL NOT NULL DEFAULT 0
);

-- hidepulsa_tokens
CREATE TABLE IF NOT EXISTS hidepulsa_tokens (
  user_id INTEGER PRIMARY KEY,
  access_token TEXT,
  refresh_token TEXT,
  expires_at INTEGER
);

-- Migrasi kolom baru ke tabel existing:
ALTER TABLE users ADD COLUMN saldo_akrab INTEGER DEFAULT 0;
ALTER TABLE pending_deposits ADD COLUMN wallet_type TEXT DEFAULT 'vpn';

15.3 Keamanan API & Webhook
Semua API key disimpan di .vars.json, tidak hardcode di source
Bearer token HidePulsa disimpan per user di tabel hidepulsa_tokens, di-refresh otomatis
Webhook HidePulsa: verifikasi HMAC SHA-256 (X-Webhook-Signature: v1={hex}), window 300 detik
Webhook Akrab: validasi format pesan via regex sebelum diproses
Admin endpoint hanya dapat diakses oleh user_id yang ada di ADMIN_IDS
Reseller endpoint hanya dapat diakses oleh user_id yang ada di ressel.db

17. Error Handling & Logging
17.1 Strategi Error Handling
Sumber Error
Penanganan
Output ke User
API Tunnel timeout
Try-catch + 30 dtk timeout; saldo tidak dipotong
'Gagal membuat akun, coba lagi nanti'
Saldo tidak cukup
Validasi sebelum panggil API; transaksi dibatalkan
'Saldo Anda tidak mencukupi. Kekurangan: Rp X'
API HidePulsa 403
Re-request OTP; clear token lama
'Sesi berakhir, silakan verifikasi ulang OTP'
Webhook format invalid
Regex mismatch → log warn + return 200
Tidak ada notif ke user; transaksi tetap pending
DB error
logger.error + reply error ke user
'Terjadi kesalahan internal, hubungi admin'
Payment expired
cleanup deposit expired saat bot start & periodik
'Pembayaran expired, silakan ulangi top up'

17.2 Logging
Logger terstruktur (winston/pino atau console terformat) untuk PM2 log
Level: info (transaksi sukses), warn (format tak dikenal), error (exception)
Log disimpan ke ./logs/out.log dan ./logs/err.log
Backup log dikirim ke chat admin via Telegram saat backup database

17.3 Notifikasi Admin Otomatis
Proyeksi bandwidth server melebihi limit → notif ke grup admin
Reseller downgrade otomatis → notif ke admin + user bersangkutan
Deposit expired / gagal → cleanup otomatis tanpa notif ke admin
Akun dihapus (self-delete / admin delete) → notif ke grup notifikasi
